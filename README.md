# CineFlow AI

> A next-generation Interactive Stateful AI Video Experience Platform

CineFlow AI creates living video experiences where AI remembers every viewer interaction, every generated scene, every decision, and continuously adapts the narrative, characters, world state, camera movement, audio, and gameplay in real time.

Think of combining Netflix Interactive, Unreal Engine, Runway, ElevenLabs, OpenAI, and game engines into one production-grade system.

## Features

- **AI-Generated Movies & TV Shows** — Entirely generated by AI with full narrative control
- **Interactive Experiences** — Every viewer interaction changes the story
- **Persistent Memory** — AI remembers every interaction and uses it to adapt future scenes
- **Character AI** — Digital humans with long-term memory, personality, goals, relationships
- **Multi-Agent Collaboration** — Director, writer, animator, camera operator agents work together
- **Real-Time Streaming** — Low-latency, adaptive bitrate video streaming
- **State Management** — Event sourcing, snapshots, rollback, and time travel
- **Procedural Worlds** — Infinite, persistent ecosystems with dynamic ecosystems
- **Video Generation** — Support for OpenAI, Runway, Luma, Pika, Stable Diffusion, custom models
- **Enterprise Architecture** — Kubernetes-ready, horizontally scalable, fault-tolerant

## Quick Start

### Prerequisites

- Docker & Docker Compose
- Node.js 18+
- Python 3.11+
- Rust 1.70+
- PostgreSQL 15+
- Redis 7+

### Development Setup

```bash
# Clone the repository
git clone https://github.com/ChaitanyaJoshi1769/CineFlowAI.git
cd CineFlowAI

# Install dependencies
./scripts/setup/install-all.sh

# Start infrastructure
docker-compose -f backend/docker-compose.yml up -d

# Start backend services
./scripts/setup/start-backend.sh

# Start frontend
cd frontend && npm install && npm run dev

# Open http://localhost:3000
```

### Production Deployment

```bash
# Deploy to Kubernetes
./scripts/deploy/deploy-prod.sh

# Monitor with Grafana
# Access at http://grafana.cineflow.ai
```

## Configuration

All services use environment variables for configuration. See `.env.example` files in each service directory.

### Global Settings

```env
# API Configuration
API_HOST=0.0.0.0
API_PORT=8000
API_LOG_LEVEL=info

# Database
DATABASE_URL=postgresql://user:pass@localhost/cineflow
REDIS_URL=redis://localhost:6379
NEO4J_URL=neo4j://localhost:7687

# AI Services
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
RUNWAY_API_KEY=...
ELEVENLABS_API_KEY=...

# Authentication
JWT_SECRET=your-secret-key
AUTH_ISSUER=https://auth.cineflow.ai

# Monitoring
PROMETHEUS_ENABLED=true
JAEGER_ENABLED=true
```
